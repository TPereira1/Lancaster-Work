Getting started - learning the UNIX shell

UNIX is an operating system first developed in the 1970s, and has been under constant development ever since. UNIX is a stable, multi-user, multi-tasking system for servers, desktops and laptops. Linux is at its heart a UNIX operating system, extended with a graphical user interface. Many operating system services and all applications you interact with run on top of the UNIX OS.

Today, we’re going to learn how to use ‘the shell’, a powerful extensible text based user interface that is the ‘power user’ interface for interacting with UNIX.

We are going to use a Linux (ubuntu) which is installed on all the lab machines in Science & Technology Building. There are lots of options for installing this on your own machine. You can also connect to a ‘virtual machine in our cluster’ via http://mylab.lancaster.ac.uk using most web browsers (pick SCC Ubuntu, click and press a key to login). This means you can work on a virtual copy of the lab machines from basically anywhere!

  • For windows users, the ‘command prompt’ is somewhat similar to the UNIX shell, but the command set and features are more limited. There is a ‘power shell’ for Windows, and this is worth learning if you want to be a truly advanced windows user. The GNU Cygwin project provides a UNIX like shell and set of tools for running natively on Windows.

  • Mac users have a ‘Terminal’ application and UNIX shell bundled with Mac OS. You can install the command line developer tools, including the C compiler we’ll use in coming weeks using: xcode-select --install.

Today’s tasks

This week, we’re going to use an open source ‘Software Carpentry’ tutorial on github that nicely introduces key features of the UNIX shell.

  • https://swcarpentry.github.io/shell-novice/

This is a great introduction to the shell that goes well beyond the core set of features you’ll need on this course to get started with programming. We’d like you to ensure that you understand the following shell features to make the coming weeks easier:

  1. Click on terminal on the toolbar, which should open a window with a UNIX bash shell. Work through replicating the commands from the files and directories tutorial, then working with files.

    Check:

    1. you know where your home directory is in the file system;
    2. that you can make a folder for your files, e.g. called scc110;
    3. that you can find your home directory from the root of the file system and navigate from your ‘home’ to the root of the file system using the cd command - what is meant by cd ..;
    4. that you can create a text file using either touch or a text editor (e.g. nano), rename it, view its contents (hint: try more and less), and delete it;
    5. know where is your ‘h drive’ is mounted in the file system and how to open, copy or move files there. You’ll want to do this because ‘h’ drive files are backed up, and any files you save on the lab machines will be lost when the machine is rebuilt and upgraded.

  2. Input and output file redirection and the UNIX shell ‘pipe’ that takes the output from one program, and ‘provides it’ as input into another program as a chain. This is incredibly powerful. We can combine programs together to achieve a task really efficiently using relatively basic parts!

    Work through the pipe and filter tutorial.

    Check:

    1. that you know how to capture the output of one program into a file and can view the captured file (for example use ls to list a folder’s contents and save this as a text file);
    2. can view this output file using cat, more and less (try and compare all 3!);
    3. can run a command (such as ls) and pipe this into less (less --help may be useful).

  3. Run man ls. Man is the manual, which includes information pages on the commands and applications bundled as part of the UNIX OS. Press h for keystroke help in the manual. Learn about common ls optional parameters, such as -a.

By the end of the lab you should be feeling confident in using the UNIX shell and basics of bash. Listing your current folder, changing folder to the parent, and back. Plus the basics of copying, renaming, and deleting files using the shell.

Hacker edition

  1. Learn about grep, an application that searches for a pattern (possibly using regular expressions) in one or more files, this is fantastically useful for searching through files, or parsing the output of an application and looking for patterns using a pipe.

  2. Using find, see if you can locate the binary executable for ls. What else is in the folder containing ls?

  3. Complete the tutorial on loops. What is the importance of history? What is meant by !! in the bash shell?
